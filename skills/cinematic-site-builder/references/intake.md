# Intake

Ask only what changes execution. Prefer defaults when the user is flexible.

## Required

- Subject: product, studio, portfolio, event, campaign, or case study.
- Page count: 1, 2, or 3.
- Sections: hero, proof, features, work/gallery, case study, CTA, contact.
- Assets: product images, video, 3D model, logo, none/procedural.
- Target: desktop-first, balanced, or mobile-first.
- Dependency permission: may Codex install npm packages if missing?

## Defaults

- Template: `vanilla-three-vite`.
- Starter: `ai-product`.
- Motion: desktop cinematic, mobile reduced but complete.
- Presets: `particle-field`, `modular-interface`, `portal-transition`.
- Fallback: Canvas 2D plus readable DOM content.
- Multi-page: use `studio-case` when the user asks for a homepage plus case study/detail page.
- Premium demo: use `lusion-lab` when the user asks for the strongest bundled Lusion-like sample.

## Do Not Ask

- Shader details unless the user is technical and asks for them.
- Exact scroll math unless replacing the runtime.
- Full copy if a draft can be generated and edited.
